The Display

100 Pixels Wide

A tall, narrow window into the device’s soul. 100×310 pixels, rendered by LVGL v9 in 8-bit indexed color with a 256-entry palette. Every wallpaper is exactly 32,036 bytes.

Dimensions

Two Displays, Two Shapes

The k·no·b·1 and Nomad E have different display geometries. The knob’s screen is unusually tall and narrow — a 10:31 aspect ratio.

Binary Format

Anatomy of a Wallpaper

Wallpapers are stored on the device as wallpaper_bg.bin in LVGL v9 image format. The format uses 8-bit indexed color (I8): each pixel is a single byte that indexes into a 256-color palette stored in the file header.

Total: 12 + 1,024 + 31,000 = 32,036 bytes

Color

256 Colors, Chosen Well

The WASM converter quantizes any input image down to a 256-color palette using perceptual color distance. Each palette entry is 4 bytes of ARGB8888. The result is surprisingly good for 8-bit color — the limited palette is a non-issue at 100 pixels wide.

Under the Hood

A Full Widget Toolkit, Mostly Locked

Firmware analysis reveals a surprisingly rich LVGL v9 stack compiled into the device. The hardware is capable of far more than the current RPC interface exposes.

Compiled Widgets

11 LVGL widgets are linked into the firmware, along with both layout engines:

WidgetTypeNotes
lv_imageDisplayStatic & tiled images, binary format only (no PNG/GIF decoding)
lv_labelDisplayText rendering with set_text_fmt for dynamic values
lv_arcDisplayCircular arc / gauge — ideal for rotary knob visualization
lv_chartDisplayData charting with get_point_pos_by_id
lv_spanDisplayRich text with mixed styles
lv_menuNavigationPage-based menu system
lv_dropdownInputSelection dropdown with options
lv_rollerInputScrollable roller selector
lv_textareaInputText input with add/delete char support
lv_buttonmatrixInputGrid of buttons
lv_calendarInputDate picker with header arrows

Typography

Text uses stock LVGL Montserrat — anti-aliased vector type, not bitmap. Confirmed via firmware string analysis (font_montserrat_14 is the bound font; other sizes are theme-resolved via theme_get_font_small/_normal/_large). The SDK exposes three slots as wlsdk.ui.FONT.BIG / MEDIUM / SMALL.

Animation & Rendering

The animation system is compiled in with property animation, style transitions, and screen transitions. The rendering stack supports blending to RGB565, RGB888, and ARGB8888 with gradient support, radius/angle masking, and vector drawing.

Animation

lv_anim_start with linear, ease-in/out, bounce, overshoot paths. Style transitions on state change (pressed, focused). Screen load animations (fade, slide) via scr_load_internal.

Layout

Both Flex and Grid layout engines are compiled in. Object tree with parent/child relationships, z-ordering, and layer management (top, system, bottom layers).

Rendering

Software renderer with gradient calculation, radius and angle masking, vector drawing, and image compositing. Multi-format blending support.

Input

The rotary encoder is registered as an LVGL input device via lv_event_get_rotary_diff. Focus groups enable keyboard-style navigation of widgets.

The gap: Almost none of these capabilities are exposed through the JSON-RPC interface. Today, the host can only upload static wallpapers and set an accent color. Animated, dynamic, or interactive screens require firmware modification.

What “Shipped” Means Here

The widgets above are written in C as native firmware screens — not SDK Python. The app’s widget-showcase UI displays them because the firmware renders them when activated. For the k·no·b·1, this matters: porting these widgets to the knob requires a firmware release, not just an app update, because they’re compiled into the device.

The coming MicroPython SDK changes this: user-written app.py widgets are Python scripts running on top of the same LVGL layer that the C widgets use. Once the SDK ports to the knob, anyone can author widgets without touching firmware — and Work Louder has publicly stated that third-party widget submissions will eventually be accepted.

The SDK

There’s a MicroPython SDK (On the Device Itself)

Work Louder ships a MicroPython SDK in the Nomad E firmware. User-written app.py runs directly on the ESP32-S3 with full access to LVGL — draw any widget, respond to encoder ticks in real time, animate at the display’s refresh rate. An optional host-side worker.py extends it for HTTP and subprocess calls. Not yet on the k·no·b·1, but the firmware codebase is shared.

Going Further

Paths to Dynamic Content

Today, on stock k·no·b·1 firmware, you can change the accent color and upload a static wallpaper. That’s the extent of host-addressable display control. To get more — animations, custom widgets, live data — you have four paths.

1

Wait for the SDK Port

The MicroPython SDK already exists on the Nomad E. Stock k·no·b·1 firmware does not ship with MicroPython — the SDK is a separate build, not a hidden capability. But the codebase is shared (nomad-e-fw) and the SDK integration (src/sdk/, custom_widget screen, MicroPython embed) is modular. A knob SDK firmware release is entirely feasible — it’s a product-roadmap question rather than an engineering one. Highest expected value if and when it ships.

2

Flash the Nomad E SDK Firmware (Experimental)

The SDK firmware exists as a public binary. Flashing it to a k·no·b·1 could give you on-device Python immediately — but display dimensions, encoder wiring, and pin assignments differ. Expect a non-functional device until you work out the deltas or wait for an official port. Recoverable via bootloader.

3

Wallpaper Cycling Hack

Rapidly upload wallpapers via fs.writebin to fake animation. ~2.5 FPS theoretical max (32 KB ÷ 4 KB chunks × 50ms delay). Viable for slow status displays and dashboard content. Works on stock firmware, no firmware changes needed.

4

Custom Firmware From Scratch

Full control. ESP32-S3, PlatformIO, no secure boot, no flash encryption. Full LVGL access — animations, widgets, arc gauges following the knob, smooth 60fps. Display controller identified as ST7789 (Ghidra analysis) — so a TFT_eSPI User_Setup.h is now a matter of #define ST7789_DRIVER plus the SPI pin map (still pending physical inspection). Longest path, highest ceiling.

Try It

Preview Without a Device

The knob-cli tool includes an offline preview command that converts any image through the full LVGL pipeline and back: 

node index.js wallpaper-preview photo.jpg preview.png
