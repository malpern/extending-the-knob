Configuration

Making It Yours

Keymaps, layers, profiles, encoders, multi-actions, BLE slots. Everything the k·no·b·1 does is driven by a single JSON file stored on the device filesystem — keymap.json.

Storage

The Keymap File

All device configuration lives in keymap.json, stored on the k·no·b·1’s LittleFS partition. It can be read and written over USB using the fs.readbin and fs.writebin RPC methods (see the Protocol page).

The file defines profiles, layers, encoder bindings, multi-actions, and macros. Editing this file is how you customize every aspect of the device’s behavior.

Schema Verified from Firmware

The schema shown below is not inferred. We extracted both default keymap JSON blobs embedded in the v0.3.2-rc.19 firmware binary (offsets 0x10735 and 0x1104d) and parsed them as JSON. The structure is real; every field name and nesting shape matches production firmware exactly.

Two variants are embedded: a 3-encoder configuration and a 2-encoder configuration. Both ship 6×14 keymap grids intended for the full-keyboard devices (Nomad E, Creator Micro v2). The device’s eFuse vendor data selects which default to load at first boot; user customizations then overwrite the file.

Keycodes

Custom Keycodes

Beyond the standard HID keycodes (KC_*), the k·no·b·1 firmware defines custom keycodes prefixed with KI_ and KA_ for device-specific functions.

Behavior

Multi-Actions

Multi-actions let a single key trigger up to four different keycodes depending on how it is pressed. Each multi-action defines handlers for four distinct event types, and can be assigned to any button or encoder via the KA_M0–KA_M3 keycodes.

●

Tap

Quick press and release.
kcOnTap

▬

Hold

Press and hold past tapping term.
kcOnHold

●●

Double-Tap

Two quick taps in succession.
kcOnDoubleTap

●▬

Tap-Hold

Tap once, then press and hold.
kcOnTapHold

The tapping term determines the threshold between a tap and a hold. The default is 250ms. A press shorter than the tapping term registers as a tap; longer registers as a hold. Double-tap detection uses the same window. 

{
  "id": "ma_01",
  "name": "Smart Play",
  "kcOnTap": "KC_MPLY",         // tap: play/pause
  "kcOnHold": "KC_MNXT",        // hold: next track
  "kcOnDoubleTap": "KC_MPRV",   // double-tap: previous track
  "kcOnTapHold": "KI_FP"        // tap-hold: function pad
}
Input

Encoders

The rotary encoders have per-layer configuration. Each encoder defines three actions: clockwise rotation, counter-clockwise rotation, and push button tap. These are defined in the layout.encoders array of each layer.

TAP CW → ← CCW
CW Clockwise rotation keycode
CCW Counter-clockwise rotation keycode
TAP Push button keycode 
"encoders": [
  {
    "cw": "KC_VOLU",    // clockwise: volume up
    "ccw": "KC_VOLD",   // counter-clockwise: volume down
    "tap": "KC_MUTE"    // push: mute
  }
]

Because encoder bindings live inside each layer, you can have completely different encoder behavior per layer — volume control on the base layer, zoom on a design layer, scroll on a code layer, and so on.

Power

Power Management

The k·no·b·1 manages power through six progressive states. As inactivity increases, the device steps down through lower power modes. Any user activity — key press, encoder turn, touch — resets the device back to the active state. Timeouts between states are configurable.

TURBO Maximum performance. Full display brightness, fastest response.
NORMAL Standard operating mode. Default state after activity.
ECO Reduced brightness and polling rate to conserve power.
STANDBY Display dimmed. Minimal activity monitoring.
SLEEP Display off. Wake on physical input only.
OFF Fully powered down. Manual power-on required.

State transitions: TURBO → NORMAL → ECO → STANDBY → SLEEP → OFF. Activity at any state returns to the current active level.

Connectivity

BLE Multi-Device

The k·no·b·1 supports up to 8 paired Bluetooth devices. Each device occupies a numbered slot. Switch between paired devices instantly using the KI_CBT1–KI_CBT8 keycodes, or initiate pairing with KI_CBTP1–KI_CBTP8. Use KI_CUSB to switch back to the USB connection.
