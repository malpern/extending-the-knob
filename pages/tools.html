Tools

Two Tools We Built

knob-cli is the control surface — a CLI for discovering, connecting to, querying, and commanding the k·no·b·1. worklouder-probe is the test surface — a ten-phase validation harness that runs on a real device and emits a structured report. Together they cover both day-to-day operation and empirical protocol verification.

Second Tool

worklouder-probe

A companion validation harness built for hands-on testing on a real device. Where knob-cli is the control surface, worklouder-probe is the test surface. It runs ten phases in sequence — discovery, USB descriptor dump, identity, filesystem inspection, protocol validation, wallpaper round-trip (with automatic backup and restore), performance timing, SDK detection probe, keymap backup, and Bluetooth scan — and writes a structured markdown report.

Non-destructive by default: the one phase that touches device state (the wallpaper round-trip) backs up the original before uploading the test image and restores it after. A --no-upload flag disables even that, making it fully read-only.

What Each Phase Validates

PhaseValidates
1. DiscoveryVID 0x303A, PIDs 0x5555/0x8296/0x82E3, HID usage page 0xFF00
2. USB DescriptorReal HID report descriptor, endpoint layout, enumerated interfaces (via system_profiler)
3. Identitysys.version, device.status, ui.active_screen, sentry.get
4. Filesystemfs.list with checksums — reveals files we don’t know about
5. Protocol ValidationProbes 10 RPC methods, records exact responses and error codes
6. Wallpaper Round-TripFull HID transport + chunked binary protocol + LVGL format — byte-for-byte verified
7. Performance20-sample RPC latency, throughput, fs.list timing
8. SDK ProbeTests 5 wlsdk.* method variations — confirms whether MicroPython runtime is active
9. Keymap BackupSaves keymap.json locally, validates schema
10. BluetoothmacOS BT pairing info scan for any Work Louder devices

How It’s Meant To Be Used

Clone the repo, npm install, node probe.mjs. After about 5 minutes, it produces output/report-{timestamp}.md with every finding in structured form. The README includes a short “For AI agents” section explaining how the tool is meant to be run by an assistant on behalf of a device owner. If every phase reports ok, our entire reverse-engineered protocol model is empirically validated. If any fails, we learn something specific and useful.
