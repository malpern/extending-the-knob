Hardware Architecture

What the Silicon Looks Like

The k·no·b·1 is built around an ESP32-S3 SoC by Espressif Systems — a dual-core Xtensa LX7 running at 240 MHz with integrated Wi-Fi and Bluetooth LE. This page maps the hardware platform, flash layout, firmware modules, and power states.

Components

Component Map

Every major subsystem on the board, from the processor to the LEDs. Each component was identified through firmware string analysis and library cross-referencing.

ESP32-S3 MCU

Dual-core Xtensa LX7, 240 MHz clock, 40 MHz crystal. Up to 16 MB flash, optional PSRAM (2 MB or 8 MB). The heart of the device.

TFT Display

100×310 pixel TFT on an SPI bus, driven by the TFT_eSPI library. Controller IC: ST7789 (identified from firmware setRotation() via Ghidra — MADCTL byte pattern and per-dimension offset values match TFT_eSPI’s ST7789 driver exactly). Rendered through LVGL v9. See the Display page for the image pipeline.

Rotary Encoders

Two rotary encoders with integrated push buttons. Used for volume, scrolling, and layer-specific actions defined in the keymap.

Mechanical Switches

Hot-swap mechanical key switches. Standard MX-compatible sockets allow swapping switches without soldering.

RGB LEDs

WS2812-compatible addressable LEDs driven via the ESP32-S3 RMT peripheral. Per-key and underglow lighting with configurable effects.

Battery System

LiPo battery with an I²C fuel gauge IC for accurate charge level reporting. Powers wireless (BLE) operation.

USB-C

TinyUSB stack exposing HID keyboard, consumer control, and vendor-specific endpoints. Also used for firmware updates and JSON-RPC communication.

Bluetooth

NimBLE stack (Apache), BLE 5.0 with HID over GATT (HOGP). Supports up to 8 paired devices with profile-based slot switching.

Flash Layout

Flash Memory Map

The 16 MB flash is divided into seven partitions. The factory application occupies the lion’s share at 8 MB. There is no OTA partition — updates are performed via the bootloader and esptool. The bars below are proportional to actual size.

Firmware

Firmware Architecture

The firmware is built with PlatformIO using Arduino-ESP32 on top of ESP-IDF 5.3.2. It was developed by Cristian Duciuc (TheCrypt0) on macOS.

Key Facts

  • — No secure boot enabled
  • — No flash encryption
  • — No OTA partition (single factory slot, updates via bootloader + esptool)
  • — Built with PlatformIO + Arduino-ESP32 + ESP-IDF 5.3.2 (custom build, Feb 10 2025)
  • — Project name nomad-e-fw — codebase shared across k·no·b·1, Nomad E, and Creator Micro v2. Cross-device string diff shows ~83% code overlap between knob-fw v0.3.2-rc.19 and nomad-e-fw v0.9.0 (1,823 of 2,179 meaningful strings shared). Device-specific divergences are primarily media player (Nomad only), encoder count, and display geometry
  • — Unstripped ELF exists internally (SHA256 recorded in app descriptor) but is not shipped publicly
  • — A parallel SDK firmware with MicroPython runtime exists (Nomad E only). The ESP32-S3 runs user-written app.py directly in MicroPython with full LVGL access. An optional host-side worker.py runs in Pyodide for HTTP and subprocess. See SDK

Power

Power Management

The wl-pm module manages six power states with configurable timeouts. Any user activity (keypress, encoder rotation, USB event) resets the device to TURBO or NORMAL. The state machine cascades downward through increasingly aggressive power savings.
