What Lies Beneath the Knob

The Work Louder k·no·b·1 is a beautifully designed keyboard with a 100×310 pixel display, dual rotary encoders, and an ESP32-S3 at its heart. This is the story of how we learned to speak its language — and what becomes possible when you do.

Methodology

How We Know What We Know

Every finding on this site was obtained through static analysis of publicly available software. No device was harmed — or even present.

1

Electron App Extraction

The official Input app (v0.14.0) is Electron-based. We extracted the app.asar archive from the macOS DMG, yielding the full JavaScript source: an 11 MB renderer bundle and a 2.5 MB main process.

2

Community Linux Port

The input-linux repository contains a community-developed port with readable, patched code — including device detection, HID communication, and RPC handling.

3

WASM Binary Extraction

The LVGL image converter is a 1.7 MB WebAssembly module embedded as base64 in the main process. We extracted it, built a JavaScript wrapper, and tested the full PNG → LVGL → PNG round-trip.

4

Firmware Binary Analysis

The v0.3.1 firmware binary (1.4 MB) was analyzed with strings and partition table parsing. This revealed: no secure boot, LittleFS at 2 MB, all linked libraries, and the full firmware module architecture.

5

GitHub Organization Inventory

Eight repositories in the Work Louder GitHub org were catalogued — including firmware releases, a node-hid fork, an experimental SDK release, and release history across four hardware products.

6

Cross-Version Firmware Diffing

Five firmware versions extracted and diffed string-by-string. Revealed actively-developed features (timer controller, WPM tracking, joystick support, BLE device abstraction) and — most significantly — a parallel SDK firmware with MicroPython v1.26.1 embedded for the Nomad E.

7

SDK Input App Extraction

The SDK-enabled Input app was extracted separately to map the host side of the architecture (the worker.py Pyodide runtime with API methods send_rpc, register_notify, handle_notify, log, exec_process).

8

Browser-Automated Doc Capture

Work Louder’s official SDK documentation is on Notion and renders client-side. We drove a headless Chromium via Playwright to capture all five sub-pages. The authoritative docs revealed the full on-device MicroPython + LVGL architecture — including lifecycle hooks, the complete wlsdk module surface, and 10 example scripts — correcting our earlier inference.
